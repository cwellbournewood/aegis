AEGIS
Authenticated · Execution · Gateway · for · Injection · Security

A personal project exploring what happens when you treat prompt injection as a structural problem instead of a content-classification one. Five composed layers, built around the idea that any defense based on filtering natural language can be bypassed by sufficiently creative natural language — so the defense should be cryptographic instead.

PL. I
Why I built it

LLMs cannot tell instructions from data.

Every defense built on natural-language filtering can be bypassed by sufficiently creative natural language. So I wanted to see what a defense built on cryptographic provenance and capability authorization would actually look like.

I
Direct Injection

Someone types "ignore prior instructions" into your input box. Classifiers can be talked around. So AEGIS doesn't ask the model to police itself.

II
Indirect Injection

A malicious email or web page reaches the model through legitimate retrieval and tells it to call a tool. The capability layer makes that structurally impossible.

III
Memory Poisoning

Injection planted in an earlier turn or persistent memory activates much later. The intent anchor catches the drift the moment it appears.

PL. II
How it works

Five composed layers. One audit trail.

Three structural layers (CCPT, Lattice, Capability Tokens) degrade slowly under adaptive attack. Two probabilistic layers (Anchor, Canary Tripwires) catch what the structural layers don't. The composition is the defense.

legitimate path · L2 user request → tool executes injection path · L0 retrieved content → blocked before tool runs
I

CCPT

HMAC-signed envelopes bind every chunk of context to its origin and trust level. The model sees clean text. The envelopes are internal to the proxy pipeline, never the prompt.
Structural
II

Lattice

Bell-LaPadula info-flow rules. Untrusted L0 content cannot authorize tool calls. No matter how persuasive the natural language, the structural rule holds.
Structural
III

Anchor

User intent is embedded once per session. Every proposed tool call is checked for cosine drift. Multi-step tasks accumulate anchors automatically so legitimate work doesn't trip the gate.
Probabilistic
IV

Canary

Per-session honeytoken instructions seeded into the system prompt. Output a canary token and you've revealed the injection. NFKC normalization defeats splitting attacks.
Probabilistic
V

Capability

Tool calls require HMAC-signed, parameter-constrained, single-use tokens the model cannot mint. Possession of the token is the authorization. No prompt can grant it.
Structural
PL. III
Who it's for

Three yes/no questions.

Three yes — AEGIS is probably useful. Two yes — run it in permissive mode for the audit trail. One yes or fewer — use simpler tools.

✓Useful for

  • LLMs that call tools that write, send, delete, deploy, or execute
  • LLMs that ingest content from sources you don't control: RAG, web fetch, MCP servers
  • Agents whose successful injection would cost more than a day of work to recover from
  • Claude Code, Cursor, or Cline setups with third-party MCP servers connected

✕Not useful for

  • Pure chatbots. No tools to authorize means the strongest layer is dormant.
  • Agents that only see content you authored yourself
  • Pre-product prototypes. Come back when you know what your agent does.
  • Anyone looking for a complete AI security stack — this is the action-authorization piece, not PII detection or jailbreak filtering
PL. IV
Install

One Docker command. One env var.

Point your existing Anthropic, OpenAI, or Google client at the AEGIS proxy URL. Mint capability tokens at the moment of user intent. That's the whole loop. 

# Run the proxy
docker run -d --name aegis -p 8080:8080 \
  -e AEGIS_MASTER_KEY="$(openssl rand -hex 32)" \
  ghcr.io/cwellbournewood/aegis:1.0.0

# Verify it's up
curl http://localhost:8080/aegis/health
open  http://localhost:8080/aegis/dashboard
from aegis.sdk import AegisClient
import anthropic

aegis = AegisClient(base_url="http://localhost:8080")

session = aegis.session.create(
    user_intent="summarize my latest invoice email",
    upstream="anthropic",
)

session.capabilities.mint(
    "read_email",
    constraints={"folder": {"kind": "eq", "value": "inbox"}},
)

claude = anthropic.Anthropic(base_url=session.proxy_url, api_key="sk-ant-...")
resp = claude.messages.create(
    model="claude-sonnet-4-5",
    max_tokens=512,
    messages=[{"role": "user", "content": "summarize my latest invoice"}],
    extra_body={"aegis": {
        "session_id": session.session_id,
        "capability_tokens": session.capability_tokens(),
    }},
)
import { AegisClient, c } from "@aegis/guard";
import Anthropic from "@anthropic-ai/sdk";

const aegis = new AegisClient({ baseUrl: "http://localhost:8080" });

const session = await aegis.createSession({
  userIntent: "summarize my latest invoice email",
  upstream: "anthropic",
});

await session.mintCapability("read_email", {
  constraints: { folder: c.eq("inbox") },
});

const claude = new Anthropic({
  baseURL: session.proxyUrl,
  apiKey: process.env.ANTHROPIC_API_KEY,
});
# Wrap third-party MCP servers
claude mcp add github "aegis mcp-wrap --policy strict -- npx @modelcontextprotocol/server-github"

# Or route Claude Code's API traffic through AEGIS
export ANTHROPIC_BASE_URL="http://localhost:8080/v1/anthropic"
claude
